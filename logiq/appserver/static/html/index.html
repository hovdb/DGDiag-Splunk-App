DGWIP

DGWIP


DGWip (Web Inspection Proxy) logs provide insight into skip candidates and URLS which may have an issue with the Digital Guardian client proxy. You can search on DGWip logs from a specific host, browser process, and time selection. Under the Skip Candidate table you can drill into each value to perform a search with that item as a filter. Any logs with a log name such as dgwip*.log will be ingested and parsed using dgwip as the sourcetype. To run a seach to see all raw data run: index=logiq sourcetype=dgwip. The DGWIP tab also provides insight into the WIP settings in config.xml. If debug logging is enabled you can also search for EventType Cassini under the DG Log tab.

Applications


Applications provide insight into all applications installed on the host. You can filter on Vendor and Host. You can drill into the Applications installed table to see the event data. You can search the raw log by entering the following in search: index="logiq" sourcetype=DIAG14

ProcessFlags


Process Flags provides the ability to search from the prcsflgs.dat on configured processflags, it also shows the inherited processflags identified through dg.log & dglog.bak.

DG Log

DG Log


DG Log provide a consolidiated way to search across dg.log and dglog.bak. In the dashboard you can see end-point available disk space, and EventTypes where you can drill in to show the event data. To search raw events from dg.log & dglog.bak type the following in search: index=logiq sourcetype=dglog
